# Сводка ЕДДС

Ежедневная сводка ЕДДС из присланного .docx → агрегированные (при необходимости
обезличенные) HTML-страницы → публикация на GitHub Pages → ссылка в чат через
бота. История за все дни копится в `history.jsonl`, по ней строится аналитика
(динамика по дням/неделям, сравнение периодов).

**Живой сайт:** https://aggasyas.github.io/svodka-eds/

## Что внутри

| Файл | Назначение |
|------|------------|
| `parse_svodka.py` | Разбор .docx-сводки → метрики |
| `metrics.py` | Извлечение метрик, история, сравнение день-к-дню |
| `render_html.py` | HTML-сводка дня |
| `render_analytics.py` | Страница аналитики: периоды, тренды, динамика |
| `render_index.py` | Индекс с навигацией |
| `build_all.py` | Сборщик всего сайта |
| `history.jsonl` | Накопительная история по дням |
| `docs/` | Готовый сайт (его и раздаёт GitHub Pages) |
| `bot/` | Боты (Telegram + MAX) и инструкции |

## Ручная сборка (без бота)

```bash
python build_all.py "svodka-2026-06-03.docx"
```

## Боты

Есть два бота, работают **параллельно** на одном сервере — кидаете .docx-сводку
в любой из них, страницы пересобираются и публикуются, в ответ приходят ссылки
на сводку / аналитику / индекс.

| Бот | Файл | Инструкция |
|-----|------|-----------|
| Telegram | `bot/svodka_bot.py` | [`bot/INSTRUKCIYA.md`](bot/INSTRUKCIYA.md) |
| MAX | `bot/svodka_max_bot.py` | [`bot/INSTRUKCIYA_MAX.md`](bot/INSTRUKCIYA_MAX.md) |

Общий API-клиент MAX — `bot/max_core.py`. Конфиг MAX-бота отдельный:
`bot/.env.max` (пример — `bot/.env.max.example`), systemd-юнит —
`bot/svodka-max-bot.service`. Telegram и MAX не мешают друг другу: у каждого
свой процесс и свой `.env`, оба пишут в одну и ту же историю и пересобирают
один и тот же сайт.

## Обезличивание

При включённом `MASK_PII` на страницах не публикуются персональные данные
(адреса, телефоны, ФИО) — только агрегированные цифры и направления.
